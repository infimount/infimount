One window for all your storage.

What Infimount does

A focused tool for moving, inspecting, and managing data across multiple storage backends.

Unified storage browser

Powered by OpenDAL

Infimount focuses on UX and workflows. OpenDAL handles the hard part – a unified API over dozens of storage backends.

1 Configure storage in Infimount
Fill in endpoints, credentials, and paths. Profiles are stored locally and can be versioned along with your dotfiles.
2 Infimount talks to OpenDAL
The Rust core uses OpenDAL to abstract away provider-specific APIs, signatures, and quirks.
3 You browse everything uniformly
Same UI for local files, object storage, key–value stores, and more. No need to re-learn each cloud console.

Getting started

The basics fit on a single screen. For anything more advanced, read the README and open an issue or PR.

Installation

1. Download the latest release for your platform.
2. On Linux, mark the AppImage / binary as executable.
3. On macOS, approve the app in Security & Privacy if needed.
4. On Windows, run the installer and pin Infimount to your taskbar.

First connection

1. Click “+” next to the storage list.
2. Pick a backend (S3, WebDAV, local filesystem, etc.).
3. Enter credentials, region / endpoint, and root path.
4. Save. Your new storage appears in the sidebar.

Day-to-day use

Use the left sidebar to switch storages, the main panel to browse objects, and the right panel to inspect metadata and content previews.

infimount.json · example config
{
  "profiles": [
    {
      "name": "work-s3",
      "backend": "s3",
      "endpoint": "https://s3.amazonaws.com",
      "region": "us-east-1",
      "bucket": "example-bucket",
      "access_key": "…",
      "secret_key": "…"
    },
    {
      "name": "local-dev",
      "backend": "fs",
      "root": "/home/you/dev"
    }
  ]
}
