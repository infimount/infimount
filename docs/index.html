Infimount Desktop
Your storage, your machine. No hosted Infimount backend is required. Credentials and registry files live under ~/.infimount.

Built for people who move between storage systems.

Infimount keeps the daily file-browser workflow simple while using OpenDAL underneath for backend reach.

01

Unified browser

Use grid and list views, previews, navigation, and drag-and-drop across local and cloud-backed storage sources.

02

Local-first control

Storage registry and MCP settings are stored locally. Mark storages as read-only or hide them from MCP exposure.

03

Agent-safe MCP

Expose filesystem-style MCP tools over stdio or local HTTP, with bearer auth, scoped sessions, and tool-level controls.

MCP for local AI workflows

Connect clients to the same storage sources you use in the desktop app, with explicit control over what is exposed.

  • Stdio and Streamable HTTP transport.
  • Bearer-token auth for headless HTTP mode.
  • Disable sensitive storage-management tools when not needed.
  • Version-aware tools where the backend supports object versions.

Client snippet

For local HTTP mode, keep the server bound to loopback unless you intentionally expose it. 

{
  "mcpServers": {
    "infimount": {
      "url": "http://127.0.0.1:7331/mcp"
    }
  }
}

Install with Homebrew

brew tap infimount/infimount
brew install infimount
brew install --cask infimount

Linux uses the formula. macOS can use the cask. Upgrade with brew update and brew upgrade.

First run notes

  • Linux AppImage: run chmod +x Infimount-*.AppImage before launch.
  • macOS: drag the app to Applications, then right-click and choose Open on first launch if Gatekeeper warns.
  • Windows: if SmartScreen appears, choose More info then Run anyway.
